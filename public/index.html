PlaylistAI

Descubra novas músicas automaticamente com base no seu gosto musical

Descrição

O PlaylistAPI é uma aplicação web que utiliza a API do Spotify para criar playlists personalizadas. O usuário informa algumas músicas de seu gosto, e o sistema, utilizando uma LLM, gera automaticamente uma playlist com faixas semelhantes em estilo, ritmo e gênero.

Funcionalidades Principais

  • Login com conta Spotify (OAuth)
  • Busca de músicas pelo nome ou artista
  • Geração automática de playlist baseada nas músicas selecionadas
  • Visualização das músicas da playlist
  • Salvar playlists geradas diretamente na conta do usuário

Tecnologias Utilizadas

  • Frontend: React e Tailwind CSS
  • Backend: Node.js, Express
  • API: Spotify Web API
  • Autenticação: OAuth 2.0
  • Banco de Dados: PostgreSQL

Questões

1. Quais são as principais funcionalidades que o sistema deve oferecer para atender às necessidades dos usuários?

O sistema deve oferecer funcionalidades que permitam ao usuário descobrir novas músicas com base em seu gosto musical e, opcionalmente, integrar essa playlist à sua conta do Spotify. As principais funcionalidades são:

  1. Cadastro e autenticação de usuários
    • Permitir que o usuário crie ou acesse uma conta usando e-mail/senha ou login social.
    • Alternativamente, possibilitar login direto com Spotify.
  2. Pesquisa de músicas em tempo real
    • O usuário pode digitar o nome da música, artista ou álbum.
    • A aplicação consulta a API do Spotify e exibe resultados instantâneos.
  3. Seleção das músicas preferidas
    • O usuário escolhe 4 músicas de seu gosto.
    • O sistema envia essas músicas até o backend.
  4. Geração de playlist personalizada
    • Envia as 4 músicas escolhidas para uma IA/LLM.
    • A IA retorna uma playlist de músicas com perfil semelhante ou complementar.
  5. Visualização da playlist gerada
    • O usuário pode ver todas as músicas sugeridas antes de exportar.
    • Possibilidade de remover músicas indesejadas antes de finalizar.
  6. Exportação para o Spotify (opcional)
    • Caso o usuário conecte sua conta ao Spotify, o sistema mostra um botão para exportar a playlist para a conta pessoal.
  7. Histórico de playlists geradas
    • Usuários logados podem visualizar playlists anteriores criadas pela IA.
    • Podem recuperar e exportar playlists antigas.
  8. Independência para usuários sem Spotify
    • Mesmo sem login do Spotify, o usuário pode gerar playlists e visualizar resultados.
    • Apenas a exportação fica restrita à conta vinculada.

2. Quais características de qualidade (como desempenho, segurança, usabilidade ou escalabilidade) são importantes para o seu sistema desenvolvido?

O sistema proposto deve apresentar um conjunto de características de qualidade que garantam uma boa experiência ao usuário e o funcionamento confiável das integrações externas, especialmente com o Spotify e com o serviço de IA. As principais são:

  1. Usabilidade
    • A interface deve ser simples, intuitiva e responsiva, facilitando a busca, seleção e geração de playlists.
    • O fluxo de uso — desde a inserção das músicas até a exportação da playlist — deve ser direto e agradável, mesmo para usuários sem conhecimentos técnicos.
  2. Desempenho
    • As respostas às buscas por músicas devem ser rápidas, com baixa latência nas requisições à API do Spotify.
    • A geração da playlist pela IA deve ocorrer em tempo razoável, evitando esperas longas que prejudiquem a experiência do usuário.
  3. Segurança
    • O sistema deve proteger os dados dos usuários, especialmente tokens de acesso obtidos pelo login via Spotify.
    • A autenticação deve seguir padrões OAuth 2.0, garantindo que o usuário controle quais permissões concede.
    • Todas as comunicações com o servidor e APIs externas devem ocorrer via HTTPS.
  4. Escalabilidade
    • A arquitetura deve suportar o aumento de usuários simultâneos sem perda significativa de desempenho.
    • O backend deve ser estruturado de forma modular, facilitando a adição de novas integrações (como outras plataformas musicais além do Spotify).
  5. Confiabilidade
    • O sistema deve ser estável, mantendo o funcionamento adequado mesmo em caso de falhas temporárias nas APIs externas.
    • Deve haver tratamento de erros e mensagens claras ao usuário em caso de falhas na geração da playlist ou exportação.
  6. Manutenibilidade
    • O código deve seguir boas práticas de organização e documentação, facilitando futuras atualizações.
    • A integração com APIs e serviços externos deve ser isolada em módulos independentes, para permitir substituições ou melhorias sem impactar o restante do sistema.
